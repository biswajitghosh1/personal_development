Introduction

Assignment groups are the groups which are created / used to deploy applications, profiles, sensors and also used for excluding any of these. The Assignment groups also known as "Smart Groups", can either be MDM based or Directory based. Directory based Smart groups are linked to Active Directory (FGO and FGR groups) and any changes to these groups in Active Directory will be replicated in MDM based on the sync which is set to once every 24 hours. The sync can also be done manually and the access is retained by the Workspace Admins. This document covers the smart groups which is in Production as per the following platforms:

All Smart groups which are created need to follow a particular naming convention which is: (platform)_(reason_for_the_group)_(name_of_the_group).There should not be any "space" in the name of the group and all spaces needs to be replaced with an underscore "_".
